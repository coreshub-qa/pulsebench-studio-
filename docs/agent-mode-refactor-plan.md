│ 现有 AgentModePage.tsx 已实现所有业务逻辑（API 调用、状态管理、表单持久化均正确），但交互动线严重混乱：                                                                                                            │
│                                                                                                                                                                                                                    │
│ - 19 个表单字段堆砌在左侧三个 SectionCard 里，没有优先级区分                                                                                                                                                       │
│ - 右侧 360px 固定栏里塞了"控制台"、草案卡片、Run 卡片、假设/风险——全部纵向堆叠，空间极度拥挤                                                                                                                       │
│ - 草案生成后，结果仍在同屏右侧小栏里呈现，用户无法清楚"现在该做什么"                                                                                                                                               │
│ - 没有任何阶段感：输入、生成、审阅、执行四步完全混在一起                                                                                                                                                           │
│ - 执行按钮在页面加载时就显示（虽然 disabled），但与生成按钮并排，语义模糊                                                                                                                                          │
│                                                                                                                                                                                                                    │
│ 重构目标：在不改动任何后端逻辑、不改动 API 调用代码的前提下，重新设计页面布局和交互状态机，使用户一次只看到一个决策点。                                                                                            │

│ 重构策略                                                                                                                                                                                                           │
│                                                                                                                                                                                                                    │
│ 阶段状态机（替换原来的 loading/executing bool）                                                                                                                                                                    │
│                                                                                                                                                                                                                    │
│ idle → planning → planned → executing → done                                                                                                                                                                       │
│                                                                                                                                                                                                                    │
│ - idle：输入阶段，显示表单                                                                                                                                                                                         │
│ - planning：生成中，全屏 loading overlay（不允许编辑）                                                                                                                                                             │
│ - planned：审阅阶段，隐藏表单，展示草案全屏                                                                                                                                                                        │
│ - executing：执行中，禁用所有操作                                                                                                                                                                                  │
│ - done：导航到 /batch/:batchId（页面卸载）                                                                                                                                                                         │
│                                                                                                                                                                                                                    │
│ 布局变化                                                                                                                                                                                                           │
